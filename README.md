# 智能农场监控系统 - STM32 FreeRTOS 教学例程

## 项目简介

本项目是一个基于STM32F103和FreeRTOS的智能农场监控系统教学例程，旨在帮助学习者理解嵌入式实时操作系统的应用开发。系统通过多个传感器实时监测农场环境参数（温度、湿度、土壤湿度、光照强度、降雨量），当环境参数超出安全范围时触发报警，并通过OLED显示屏展示数据，支持用户通过按键和旋钮进行交互操作。

本课程设计版本已扩展为“STM32 + LoRa + ESP32-S3 + MQTT + Node-RED + Android”完整链路。硬件接好后，优先按 [联调步骤.md](联调步骤.md) 操作。项目总览可查看 [PROJECT_SUMMARY.md](PROJECT_SUMMARY.md)。

## 硬件平台

- **主控芯片**：STM32F103C8T6
- **操作系统**：FreeRTOS（CMSIS-RTOS2接口）
- **显示屏**：OLED 128x64（I2C接口）
- **传感器**：
  - AHT20温湿度传感器（I2C接口）
  - 土壤湿度传感器（ADC接口）
  - 光照强度传感器（ADC接口）
  - 降雨量传感器（ADC接口）
- **输入设备**：
  - 按键KEY1：页面切换
  - 按键KEY3：阈值编辑模式切换
  - 旋钮：浏览/编辑阈值
- **通信接口**：
  - UART3：E32-433T20D LoRa模块通信
  - I2C1：OLED和AHT20共享总线
- **输出设备**：
  - 蜂鸣器（TIM4 PWM控制）

## FreeRTOS资源详解

### 任务（Tasks）

系统共创建了4个任务，每个任务负责不同的功能模块：

#### 1. SensorTask（传感器任务）

- **功能**：传感器数据采集和报警检测
- **优先级**：`osPriorityNormal`（普通优先级）
- **堆栈大小**：1024字节（256 * 4）
- **执行周期**：1000ms（1秒）
- **主要工作**：
  1. 初始化所有传感器（AHT20、土壤湿度、光照、降雨量）
  2. 定期读取传感器数据并更新全局状态变量 `farmState`
  3. 检查各环境参数是否超出安全范围（`farmSafeRange`）
  4. 当检测到异常时：
     - 启动蜂鸣器（调用 `Beep_on()`）
  5. 无异常时关闭蜂鸣器（调用 `Beep_off()`）

- **关键代码位置**：`Core/App/Tasks/SensorTask.c`

#### 2. InputTask（输入任务）

- **功能**：用户输入处理（按键、旋钮）
- **优先级**：`osPriorityHigh`（高优先级，保证用户输入响应及时）
- **堆栈大小**：512字节（128 * 4）
- **执行周期**：10ms（快速响应）
- **主要工作**：
  1. 检测KEY1按下：切换页面（首页 ↔ 阈值设置页）
  2. 在阈值设置页时：
     - 检测KEY3按下：切换浏览/编辑模式
     - 检测旋钮旋转：
       - **浏览模式**：切换选中的阈值项（左旋/右旋）
       - **编辑模式**：修改当前选中项的值（左旋减小/右旋增大）
  3. 修改阈值时进行边界检查，确保数值合理

- **关键代码位置**：`Core/App/Tasks/InputTask.c`

#### 3. ScreenTask（显示任务）

- **功能**：OLED屏幕显示
- **优先级**：`osPriorityBelowNormal5`（低优先级，不影响实时性要求高的任务）
- **堆栈大小**：512字节（128 * 4）
- **执行周期**：10ms（保证显示流畅，100Hz刷新率）
- **主要工作**：
  1. 初始化OLED显示屏
  2. 根据当前页面索引（`pageIndex`）渲染相应界面：
     - **首页（PAGE_HOME）**：显示当前环境状态数据
       - 第一行：温度、湿度、光照强度
       - 第二行：土壤湿度、降雨量、水泵状态
     - **阈值设置页（PAGE_RANGE）**：显示和编辑报警阈值
       - 第一页：温度、湿度、光照强度的阈值
       - 第二页：土壤湿度、降雨量的阈值
  3. 在阈值设置页中：
     - 显示当前选中的阈值项（下划线指示）
     - 编辑模式下，下划线闪烁（每10次刷新切换一次）
     - 浏览模式下，下划线固定显示
  4. 使用互斥锁保护I2C总线，将帧缓冲区内容发送到OLED

- **关键代码位置**：`Core/App/Tasks/ScreenTask.c`

#### 4. LoRaTask（LoRa通信任务）

- **功能**：LoRa通信（通过UART3发送农场数据）
- **优先级**：`osPriorityLow`（低优先级，不影响实时性要求高的任务）
- **堆栈大小**：512字节（128 * 4）
- **执行方式**：每秒周期发送一次
- **主要工作**：
  1. 周期读取最新农场状态
  2. 组装JSON数据帧
  3. 通过UART3发送到E32-433T20D模块
  4. 由对端网关转发到MQTT/Web/Android

- **关键代码位置**：`Core/App/Tasks/lora_e32.c`

### LoRa链路

- **用途**：STM32通过E32-433T20D向ESP32网关周期发送农场数据
- **消息格式**：JSON字符串
  ```json
  {"temp":25.6,"hum":68.0,"soil":42,"light":520,"rain":0,"pump":1,"alarm":0}
  ```
- **使用流程**：
  1. SensorTask更新 `farmState`
  2. LoRaTask每秒采样一次最新状态
  3. 通过UART3发送JSON到E32模块
  4. ESP32网关接收后上传MQTT

- **关键代码位置**：
  - 创建：`Core/Src/freertos.c`
  - 发送：`Core/App/lora_e32.c`

### 软件定时器（Timer）

#### BeepTimer

- **用途**：控制蜂鸣器的周期性响铃
- **类型**：周期性定时器（`osTimerPeriodic`）
- **周期**：500ms（在 `Beep_on()` 中通过 `osTimerStart` 设置）
- **回调函数**：`BeepTimerCallback`
- **工作原理**：
  1. 当 `Beep_on()` 被调用时，启动定时器
  2. 定时器每500ms触发一次回调函数
  3. 回调函数在开启和关闭状态之间切换：
     - **状态0**：开启PWM，设置占空比100（产生声音）
     - **状态1**：关闭PWM，设置占空比0（停止声音）
  4. 实现500ms开、500ms关的周期性响铃效果
  5. 当 `Beep_off()` 被调用时，停止定时器并立即关闭蜂鸣器

- **关键代码位置**：
  - 创建：`Core/Src/freertos.c` 第149行
  - 回调函数：`Core/App/Tasks/BeepTimer.c` 第14行
  - 控制函数：`Core/App/Tasks/BeepTimer.c` 第5、8行

### 互斥锁（Mutex）

#### i2c1Mutex

- **用途**：保护I2C1总线，防止OLED和AHT20同时访问I2C1造成冲突
- **使用场景**：
  1. **SensorTask读取AHT20时**：
     - 读取前：`osMutexAcquire(i2c1MutexHandle, osWaitForever)`
     - 读取后：`osMutexRelease(i2c1MutexHandle)`
  2. **ScreenTask刷新OLED显示时**：
     - 刷新前：`osMutexAcquire(i2c1MutexHandle, osWaitForever)`
     - 刷新后：`osMutexRelease(i2c1MutexHandle)`

- **为什么需要互斥锁**：
  - OLED和AHT20都使用I2C1总线进行通信
  - I2C是共享总线，同一时刻只能有一个设备使用
  - 如果不加保护，两个任务可能同时访问I2C1，导致通信失败或数据错误
  - 使用互斥锁确保同一时刻只有一个任务能访问I2C1总线

- **关键代码位置**：
  - 创建：`Core/Src/freertos.c` 第137行
  - 使用：`Core/App/Tasks/SensorTask.c` 第64、72行；`Core/App/Tasks/ScreenTask.c` 第213、215行

## 应用层代码架构

### 全局状态管理

系统使用全局变量来管理状态，这些变量在多个任务间共享：

#### 1. farmState（环境状态）

定义在 `Core/App/global/farmState.h`，存储当前从传感器读取的实时环境数据：

```c
typedef struct {
  float temperature;       // 温度（摄氏度）
  float humidity;          // 湿度（百分比，0-100）
  uint16_t rainGauge;      // 降雨量（百分比，0-100）
  uint16_t soilMoisture;   // 土壤湿度（百分比，0-100）
  uint16_t lightIntensity; // 光照强度（勒克斯 lx）
  uint8_t waterPumpState;  // 水泵状态：0-关闭, 1-开启
} FarmState;
```

- **更新者**：SensorTask（每1秒更新一次）
- **读取者**：ScreenTask（用于显示）

#### 2. farmSafeRange（安全范围阈值）

定义在 `Core/App/global/farmState.h`，定义各环境参数的安全范围阈值：

```c
typedef struct {
  float minTemperature;       // 最低温度阈值
  float maxTemperature;       // 最高温度阈值
  float minHumidity;          // 最低湿度阈值
  float maxHumidity;          // 最高湿度阈值
  uint16_t maxRainGauge;      // 最大降雨量阈值
  uint16_t minSoilMoisture;   // 最低土壤湿度阈值
  uint16_t maxSoilMoisture;   // 最高土壤湿度阈值
  uint16_t minLightIntensity; // 最低光照强度阈值
  uint16_t maxLightIntensity; // 最高光照强度阈值
} FarmSafeRange;
```

- **初始化**：`EnvSafeRange_Init()` 设置默认值
- **更新者**：InputTask（用户通过旋钮修改）
- **读取者**：SensorTask（用于报警检测）

#### 3. pageIndex（页面索引）

定义在 `Core/App/global/screen.h`，控制当前显示的页面：

```c
typedef enum {
  PAGE_HOME = 0,   // 首页：显示环境状态数据
  PAGE_RANGE = 1,  // 阈值设置页：显示和编辑报警阈值
  PAGE_End,        // 结束标志
} ScreenPage;
```

- **更新者**：InputTask（KEY1按下时切换）
- **读取者**：ScreenTask（用于决定渲染哪个页面）

#### 4. rangeEditIndex（阈值编辑索引）

定义在 `Core/App/global/screen.h`，在阈值设置页中选择要编辑的阈值项：

```c
typedef enum {
  RANGE_EDIT_TEMPERATURE_MIN = 0,      // 最小温度
  RANGE_EDIT_TEMPERATURE_MAX = 1,      // 最大温度
  RANGE_EDIT_HUMIDITY_MIN = 2,         // 最小湿度
  RANGE_EDIT_HUMIDITY_MAX = 3,         // 最大湿度
  RANGE_EDIT_LIGHT_INTENSITY_MIN = 4,   // 最小光照强度
  RANGE_EDIT_LIGHT_INTENSITY_MAX = 5,   // 最大光照强度
  RANGE_EDIT_SOIL_MOISTURE_MIN = 6,     // 最小土壤湿度
  RANGE_EDIT_SOIL_MOISTURE_MAX = 7,     // 最大土壤湿度
  RANGE_EDIT_RAIN_GAUGE_MAX = 8,        // 最大降雨量
  RANGE_EDIT_END,
} RangeEditIndex;
```

- **更新者**：InputTask（旋钮旋转时切换）
- **读取者**：ScreenTask（用于显示当前选中的项）

#### 5. rangeEditState（阈值编辑状态）

定义在 `Core/App/global/screen.h`，区分浏览模式和编辑模式：

```c
typedef enum {
  RANGE_EDIT_STATE_NORMAL = 0,   // 正常状态：浏览模式
  RANGE_EDIT_STATE_EDITING = 1,  // 编辑状态：编辑模式
} RangeEditState;
```

- **更新者**：InputTask（KEY3按下时切换）
- **读取者**：InputTask（决定旋钮的行为）、ScreenTask（决定下划线是否闪烁）

### 任务间通信

系统使用多种方式进行任务间通信：

#### 1. 全局变量（共享内存）

- **用途**：传递状态信息（环境数据、页面索引等）
- **优点**：简单直接，访问速度快
- **缺点**：需要保证数据一致性（本系统中，每个变量只有一个写入者，多个读取者，相对安全）
- **使用场景**：
  - `farmState`：SensorTask写入，ScreenTask读取
  - `pageIndex`：InputTask写入，ScreenTask读取
  - `farmSafeRange`：InputTask写入，SensorTask读取

#### 2. 数据上报

- **用途**：周期发送最新农场状态
- **优点**：实现简单，和传感器采集周期一致
- **使用场景**：
  - LoRa：LoRaTask发送实时状态，ESP32网关接收并转发

#### 3. 互斥锁

- **用途**：保护共享资源（I2C1总线）
- **使用场景**：
  - i2c1Mutex：SensorTask和ScreenTask访问I2C1前获取，访问后释放

## 功能说明

### 传感器监控

系统实时监控以下环境参数：

1. **温度**（AHT20传感器，I2C接口）
   - 单位：摄氏度（℃）
   - 默认安全范围：15-30℃
   - 更新频率：1秒

2. **湿度**（AHT20传感器，I2C接口）
   - 单位：百分比（%）
   - 默认安全范围：20-70%
   - 更新频率：1秒

3. **土壤湿度**（土壤湿度传感器，ADC接口）
   - 单位：百分比（%）
   - 默认安全范围：20-80%
   - 更新频率：1秒

4. **光照强度**（光照传感器，ADC接口）
   - 单位：勒克斯（lx）
   - 默认安全范围：30-800 lx
   - 更新频率：1秒

5. **降雨量**（降雨量传感器，ADC接口）
   - 单位：百分比（%）
   - 默认安全范围：< 80%
   - 更新频率：1秒

### 数据上报与告警

当环境参数超出安全范围时，系统会触发蜂鸣器，同时把最新状态通过 LoRa 上报给网关：

1. **采集周期**：SensorTask 每 1 秒更新一次 `farmState`
2. **告警动作**：若超限则启动蜂鸣器
3. **上报动作**：LoRaTask 每 1 秒打包最新环境数据并发送到 E32
4. **数据格式**：
   ```json
   {"temp":25.6,"hum":68.0,"soil":42,"light":520,"rain":0,"pump":1,"alarm":0}
   ```
5. **网关转发**：ESP32 网关接收后上传 MQTT，供 Web 和 Android 端订阅

### 用户交互

系统支持以下用户交互：

1. **页面切换**（KEY1）：
   - 首页 ↔ 阈值设置页
   - 循环切换

2. **阈值编辑**（在阈值设置页）：
   - **KEY3**：切换浏览/编辑模式
   - **旋钮左旋/右旋**：
     - 浏览模式：切换选中的阈值项
     - 编辑模式：减小/增大当前选中项的值
   - **编辑规则**：
     - 浮点数类型（温度、湿度）：每次调整0.1
     - 整数类型（土壤湿度、光照、降雨量）：每次调整1
     - 自动进行边界检查，确保数值合理

### 显示界面

#### 首页（PAGE_HOME）

显示当前环境状态数据：

## 学习要点

### FreeRTOS核心概念

1. **任务（Task）**：
   - 理解任务的创建、优先级、堆栈大小配置
   - 理解任务的状态转换（就绪、运行、阻塞、挂起）
   - 理解任务调度机制（抢占式调度）

2. **数据上报**：
   - 理解周期上报与共享状态读取
   - 理解如何把采集数据打包后通过串口发送

3. **软件定时器（Timer）**：
   - 理解周期性定时器和单次定时器
   - 理解定时器回调函数的执行上下文
   - 理解定时器的启动和停止

4. **互斥锁（Mutex）**：
   - 理解互斥锁的作用（保护共享资源）
   - 理解互斥锁的获取和释放
   - 理解死锁的避免

### 嵌入式系统设计

1. **分层架构**：
   - 应用层、BSP层、HAL层的划分
   - 各层之间的接口设计

2. **任务划分**：
   - 如何根据功能模块划分任务
   - 如何确定任务的优先级和周期

3. **任务间通信**：
   - 何时使用全局变量
   - 何时使用消息队列
   - 何时使用互斥锁

4. **资源保护**：
   - 共享资源的保护（I2C总线）
   - 临界区的识别和保护

### 代码阅读建议

1. **从main.c开始**：
   - 理解系统初始化流程
   - 理解FreeRTOS调度器的启动

2. **理解freertos.c**：
   - 理解所有FreeRTOS资源的创建
   - 理解任务的创建顺序

3. **逐个任务分析**：
   - 从SensorTask开始，理解数据采集流程
   - 理解InputTask的用户交互处理
   - 理解ScreenTask的显示逻辑
   - 理解LoRaTask的通信处理

4. **理解全局状态管理**：
   - 理解farmState和farmSafeRange的作用
   - 理解页面和编辑状态的管理

## 注意事项

1. **缓冲区管理**：
   - 上报数据使用固定长度缓冲区，注意避免越界
   - 字符串格式化后要确认长度足够

2. **互斥锁使用**：
   - 获取互斥锁后必须释放，避免死锁
   - 不要在中断中长时间持有互斥锁

3. **任务优先级**：
   - 合理设置任务优先级，避免低优先级任务饿死
   - 高优先级任务不应长时间占用CPU

4. **堆栈大小**：
   - 根据任务的实际需求设置堆栈大小
   - 堆栈过小会导致栈溢出，过大浪费内存
