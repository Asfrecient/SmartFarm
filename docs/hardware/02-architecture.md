# SmartFarm 课程设计报告
# 阶段 2：架构候选
# 日期：2026-06-23

## 候选架构对比

| 方案 | 组成 | 优点 | 缺点 | 适用条件 |
|---|---|---|---|---|
| 国产优先 | 国产 MCU/LoRa/WiFi/云平台替代 | 供应链自主性更好 | 教学资料和示例相对少 | 有国产替代要求 |
| 海外主流优先 | STM32 + ESP32 + EMQX + Node-RED | 生态成熟，资料多 | 部分器件依赖海外生态 | 课程周期紧、需要快速落地 |
| 混合折中 | STM32 采集 + ESP32 网关 + MQTT + Node-RED + Android | 复用现有工程，风险最低 | 方案链路较长 | 本课设最合适 |

## 本项目选择

选择“混合折中”方案。

### 理由

1. STM32 侧已有完整传感器与 FreeRTOS 代码，复用价值最高。
2. ESP32-S3 适合做 LoRa 网关和 WiFi/MQTT 转发。
3. Node-RED 可快速搭建 Web 可视化。
4. Android 端可作为答辩展示的移动监控界面。

## 系统架构

STM32F103 负责采集、控制和 LoRa 上报；
ESP32-S3 负责接收 LoRa 数据并发布 MQTT；
Node-RED 与 Android 订阅 MQTT 主题 `smartfarm/data` 展示数据。

## 结论

架构方向已确定，可进入实现与接口收口阶段。
